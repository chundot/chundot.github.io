VS Code插件黑名单——反向推荐一波插件

发表于2021-08-17|更新于2021-11-19|各种折腾
|阅读量:

不断更新的 vscode 地雷插件名单（前端为主）。

淘汰插件名单

Beautify

格式化插件。

现在前端格式化Prettier才是王道啊！！！

Debugger for Chrome

Chrome 调试 js 的插件。

大大的Deprecated，不会有人看不见吧？

npm Intellisense

补全 npm 模块的插件。

很久没更新了。七月份有更新，但没什么卵用。

vscode 目前内置 npm 模块补全的功能了，这个插件根本没有安装的必要。

vscode 内置补全演示：

但 vscode 就是不能导入 js 以外的文件…比如 css，可能需要使用其他路径补全的插件。

Kite AutoComplete AI Code

AI 补全插件。

开发团队跑路了，这个已经暂时不能用了，什么时候能用了再把它从名单里去掉。

同类型的有个Tabnine，共同点就是性能都很差，免费版都很残疾。

HTML Snippets

HTML 代码片段。

这个还会 vscode 内置的 HTML 功能冲突。

vscode 内置了emmet，这个没有什么存在的必要了。

不推荐插件名单

Bracket Pair Colorizer

括号上色插件。

为什么不推荐？因为有个Bracket Pair Colorizer 2，可能性能更好速度更快。

另外，vscode 会对光标所在位置的括号对进行高亮的，上色啥的是不是伪需求？

IntelliSense for CSS class names in HTML

自动扫描 CSS 以及补全 class 的插件。

功能类似的有个HTML CSS Support。

都不好使，都很卡。但并不是不推荐，两个里面挑个你喜欢的就行了。

Path Intellisense

路径建议补全插件。

这个插件要求你关闭 vscode 自带的路径建议，问题出在这里：

{ "typescript.suggest.paths": false }

这意味着 npm 模块补全就没了，插件主页还给你推荐npm Intellisense…

不关的话 js 文件建议会重复。

有个插件叫Path Autocomplete，可以和 vscode 自带的路径建议共存无重复，用如下的设置：

"path-autocomplete.excludedItems": {
  "**/{.git,*.js}": { "when": "**" } // always ignore .git folder and js files
}

只可惜 Path Intellisense 里我没有找到类似这样的选项…
而且看这个插件仓库的提交历史记录就知道，没有高质量更新。
